# Foundation Skills

Foundation skills are cross-cutting . they apply in any Triple Diamond phase and are not tied to a specific product-development stage. Use them to shape perspective (persona), frame strategy (lean canvas), or manage the meeting lifecycle that supports every initiative (meeting family).

## Skills in this phase

| Skill | Description | Command |
|-------|-------------|---------|
| [foundation-lean-canvas](foundation-lean-canvas.md) | One-page lean canvas across nine interlocking blocks (problem, customer, UVP, solution, channels, revenue, cost, metrics, unfair advantage) with optional HTML visual rendering | `/lean-canvas` |
| [foundation-persona](foundation-persona.md) | Evidence-calibrated product or marketing persona using the canonical v2.5 template contract | `/persona` |
| [foundation-meeting-agenda](foundation-meeting-agenda.md) | Attendee-facing meeting agenda with time-boxed topics, type tags, owners, and prep | `/meeting-agenda` |
| [foundation-meeting-brief](foundation-meeting-brief.md) | User's private strategic prep with stakeholder reads, ranked outcomes, key messages, and anticipated Q&A | `/meeting-brief` |
| [foundation-meeting-recap](foundation-meeting-recap.md) | Topic-segmented post-meeting summary with decisions highlighted and actions inline | `/meeting-recap` |
| [foundation-meeting-synthesize](foundation-meeting-synthesize.md) | Cross-meeting archaeology surfacing patterns and contradictions across recaps | `/meeting-synthesize` |
| [foundation-stakeholder-update](foundation-stakeholder-update.md) | Async stakeholder communication with channel and audience variants, tailored from a recap | `/stakeholder-update` |

## Meeting Skills Family

The five `foundation-meeting-*` skills (agenda, brief, recap, synthesize, stakeholder-update) form a coordinated family governed by the [Meeting Skills Family Contract](../../reference/skill-families/meeting-skills-contract.md). They share a common behavioral pattern (zero-friction execution with `go` mode), frontmatter schema, and filename-based chaining so artifacts compose cleanly: a recap auto-populates from an agenda, a synthesis consumes recaps, a stakeholder update translates a recap for non-attendees.

See the [family contract](../../reference/skill-families/meeting-skills-contract.md) for the shared behavioral requirements, or invoke any meeting skill individually for a single-artifact workflow.
